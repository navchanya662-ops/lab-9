# Лабораторна робота 13

## Тема

Робота з формами на клієнті: Fetch API, localStorage та інтеграція з Express API.

## Мета роботи

Мета роботи - створити клієнтську частину для онлайн-магазину, навчитися надсилати HTTP-запити з браузера до власного API, зберігати JWT-токен у `localStorage` та використовувати його для захищених запитів.

## Варіант

Варіант 1: онлайн-магазин.

Основна сутність: `Product`.

## Що зроблено

- Підключено роздачу статичних файлів через `express.static('public')`.
- Створено папку `public/`.
- Додано сторінки `register.html`, `login.html`, `index.html`, `create.html`.
- Усі запити з клієнта виконуються через чистий `fetch`.
- JWT-токен після реєстрації або входу зберігається в `localStorage`.
- При створенні товару токен передається в заголовку `Authorization: Bearer <token>`.
- Якщо токен відсутній на сторінці створення товару, користувача перенаправляє на `login.html`.
- У формах використовується `event.preventDefault()`.
- Перед обробкою відповіді перевіряється `response.ok`.

## Структура клієнтської частини

```text
public/
  index.html
  register.html
  login.html
  create.html
  styles.css
```

## Опис файлів

### `public/register.html`

Форма реєстрації з полями `name`, `email`, `password`, `confirmPassword`. Надсилає `POST /api/auth/register`, зберігає токен у `localStorage` і перенаправляє на `index.html`.

### `public/login.html`

Форма входу з полями `email` і `password`. Надсилає `POST /api/auth/login`, зберігає JWT-токен у `localStorage` і відкриває головну сторінку.

### `public/index.html`

Головна сторінка онлайн-магазину. Завантажує товари через `GET /api/products`, показує їх у картках і має кнопку виходу, яка видаляє токен.

### `public/create.html`

Форма створення товару з полями `name`, `description`, `price`, `category`, `stock`. Якщо токена немає, сторінка перенаправляє користувача на `login.html`. Якщо токен є, форма надсилає `POST /api/products` із заголовком `Authorization`.

### `public/styles.css`

Файл базового оформлення сторінок: навігація, форми, повідомлення, картки товарів.

## Висновок

У лабораторній роботі реалізовано базову клієнтську частину онлайн-магазину. HTML-сторінки взаємодіють із серверним API через Fetch API, токен авторизації зберігається в `localStorage`, а захищені запити передають його в заголовку `Authorization`.
